This is a full stack project to manage hierarchy of employees in an organization. Front end is built using Angular. Server side uses Spring Boot and data is stored using MySQL server. Route guards are implemented to prevent access without proper authorization and session id is used to track the current login session. Basic login with session id and ip address is implemented at present. Further development idea is to implement a more secure technique. Each employee has a rank associated (1 being the highest rank possible - Boss). Each rank can have rank number of employees (Only 17 employees can be of the rank 17).

Repo Link: https://github.com/Krishna-Chilukuri/Employee-Management-Portal
Following operations can be performed on this hierarchy:

Operation Description
Add Employee An Employee can be added to the hierarchy by providing an employee name and rank. The rank clause is checked and then the employee is inserted into the hierarchy. Rebalancing of the hierarchy is done to ensure that employees of rank less than that of the employee being inserted do not end up being in the same level of the tree if visualized.
Remove Employee An Employee is removed based on the employee Id. If the employee has reportees assigned, they are assigned to some colleague of same rank or the boss of the employee. If both are not present, the highest rank reportee of the reportees is promoted and becomes the boss for the remaining reportees.
View All Employees All Employees of the organization are displayed. Each employee record consists of Employee Id, Name, Rank, Reportees(Juniors) and Reports To(Boss).
View Employee A Particular Employee can be retrieved using the ID. The record consists of Employee Id, Name, Rank, Reportees(Juniors) and Reports To(Boss). Further developments are being done to search based on the name of the employee
View Hierarchies Upper, Lower and K-Step hierarchies of an employee can be viewed based on the ID of the employee.
  1. Upper Hierarchy consists of all the employees of the hierarchy that are above the current employee along with the current employee.
  2. Lower Hierarchy consists of the sub tree of the hierarchy where the current employee is the boss.
  3. K-Step Hierarchy returns the employees at a distance of k from the current employee (Boss at k distance and Juniors at k distance).
Promote Employee Employee can be promoted by k ranks if the rank clauses permit it. For each promotion,if rank clause is not violated, that particular employee is removed from hierarchy and then added back with the updated rank.
Demote Employee Employee can be demoted by k ranks if the rank clauses permit it. For each demotion,if rank clause is not violated, that particular employee is removed from hierarchy and then added back with the updated rank.
There are 4 types of privileges in this application:
  1. Guest: Users of this privilege are permitted only to view the hierarchy.
  2. Privileged User: Every employee in the hierarchy is a privileged user. They can add, remove, promote or demote employees that are below them in hierarchy. However, promotions are not possible to the same rank as that of the user.
  3. Admin: Admin users can be described as privileged users with a rank 0. So, they can modify the hierarchy however needed. An employee from the hierarchy can be promoted to Admin privilege by a Owner.
  4. Owner: Owners have all the privileges as that of Admins and can promote a user to Admin or Owner, demote an Owner and remove an Admin/ Owner. Owner exclusive operations are:
    Operation Description
    Promote to Admin An Employee can be promoted to be an Admin. When this is done, that particular employee is removed from the hierarchy. It can be described as being rank 0.
    Promote to Owner An Admin is promoted to Owner.
    Demote Owner An Owner is demoted to Admin.
    Remove Admin/ Owner Remove an Admin or Owner user from the application.
